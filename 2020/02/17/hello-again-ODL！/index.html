继续，按照官方文档，cd到example下mvn clean install，这里我推荐使用mvn clean install -DskipTests
因为这个命令可以跳过测试，省去一大部分时间(odl为什么不是热部署)

继续，按照官方文档cd下去并且执行karaf，出现了意料之外的错误

不过这不是什么大的错误，仔细查一下就会发现是jdk版本问题，我用的是新潮的13.0.2，但是镁版本需要11.x才行，问题不大，重装一遍jdk到11版，再次运行karaf，成功

继续，log: display | grep xxx 查询相关日志信，然后shutdown -f关掉odl
