气泡弹窗总结

enumerateBlock

几种主流遍历方式

stop参数的作用

顾名思义，用来作为停止遍历的标志的。
但是，在stop置为YES时，并不会马上退出遍历，而是会将block剩下的代码运行完整才会退出遍历。

可以看出，stop相当于一个并不完全的break；

return对它的影响

如果在这之中用到了return，会结束函数吗？

答案是不会的。

不仅不会结束函数，反而后续的遍历还能继续运行。
所以在这个枚举block中，return的作用相当于countine.

更进一步

既然stop是执行本轮代码之后结束循环，return是continue，那两者结合呢？
我就知道会有人这么问


可以看到，遍历会马上退出，但实际上程序并没有马上退出。
所以枚举block中的 stop + return = 其他遍历模式中的break；

block 枚举的优势

使用 block 来枚举时，block 内部会自动添加一个 autoreleasepool：

在普通 for 循环和 for-in 循环中没有，新版的 block 版本枚举器更加方便。for 循环中遍历产生大量 autorelease 变量时，就需要手动加局部 autoreleasepool。

比较汇总

  • 在进行单纯的于被便利数组无关的便利操作时
    • for in最快；for其次；enumBlock最慢
  • 在进行对便利数组读写操作的便利时
    • enumBlock最快；for in次快；for最慢
  • 在只要求速度不要求顺序的便利中
    • enumBlock可以通过option设置并行便利，大大提高速度
